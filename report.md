## Final Evaluation Metric: Hold-Out Test MAPE: 1.86%
 Project OverviewThis project builds a robust, tree-based machine learning pipeline to predict short-term power grid demand ($t+1$). By engineering chronological and macroeconomic features while strictly preventing future data leakage, the XGBoost model achieved a highly accurate 1.86% Mean Absolute Percentage Error (MAPE) on a chronologically separated, unseen test year.
 1. Data Preparation & Anomaly HandlingThe raw national grid data contained severe undocumented spikes, missing timestamps, and potential data leakage risks. Here is the rationale for how the structural integrity was maintained:Strict Chronological Alignment: All data was forced into a strict hourly grid (.resample('H')). All supply-side metrics (e.g., generation, fuel mix, load shedding) were completely dropped to prevent the model from "cheating" by learning the grid's post-facto dispatch physics.Conservative Anomaly Detection: Rather than using a standard Z-score (which gets skewed by massive outliers), a 24-hour Rolling Median and Interquartile Range (IQR) was used. A conservative multiplier of 3.0 was applied to flag extreme undocumented spikes without erasing natural, legitimate demand peaks.Organic Missing Data Handling: Flagged anomalies and missing gaps were replaced with NaN rather than interpolated. Because XGBoost utilizes a sparsity-aware split-finding algorithm, it naturally learns the best direction to route missing values. This prevents the bias that linear interpolation introduces during volatile grid events.
 2. Feature Engineering StrategyBecause classical tabular models treat observations independently, the concept of "time" had to be artificially constructed so the model could understand momentum and seasonality.Cyclical Time Encoding: Basic features like hour_of_day and month were transformed using Sine and Cosine functions. This maps time onto a continuous circle, allowing the model to mathematically understand that Hour 23 (11:00 PM) smoothly transitions into Hour 0 (Midnight) without a sudden numerical jump.Historical Lags (Backward-Looking): To capture grid momentum, lagged features were created exclusively using past data:$t-1$ and $t-2$ for immediate short-term momentum.$t-24$ to capture daily behavioral seasonality.$t-168$ to capture weekly behavioral seasonality.Rolling Volatility: 3-hour, 6-hour, and 24-hour rolling averages and standard deviations were calculated to give the model context on recent grid volatility.Macroeconomic Integration: To help the model understand long-term baseline growth, key annual macro indicators (like Industrial Value Added) were joined to the hourly data using a strict $Y-1$ lag, ensuring no future economic data leaked into the current hour's prediction.
 3. Feature Importances & Key InsightsThe model was trained using an expanding-window TimeSeries cross-validation to tune hyperparameters safely. Extracting the feature importances revealed clear, logical drivers behind national power consumption:Short-Term Momentum Dominates: The most critical drivers were demand_mw (the current hour's demand) and rolling_mean_3h. This proves the grid is highly inertial; the best predictor of consumption at 3:00 PM is how much power was being consumed at 2:00 PM.The "Shape" of the Day: Cyclical features, specifically cos_hour and sin_hour, were incredibly influential. This shows the model successfully learned human behavioral cycles (waking up, industrial shifts starting, and evening lighting peaks).Macroeconomic Baseline: Industry (including construction), value added appeared in the top most influential features. This validates the hypothesis that while weather and time drive hourly fluctuations, the fundamental scale of the grid is dictated by the nation's industrial output from the previous year.
